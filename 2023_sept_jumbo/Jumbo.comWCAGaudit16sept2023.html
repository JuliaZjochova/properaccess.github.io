WCAG audit of Jumbo.com

Research date:  3 - 16 september 2023

Report version: 1.0

Introduction

On behalf of XX, the website https://www.jumbo.com/ has been evaluated to conform to the Web Content Accessibility Guidelines (WCAG) 2.1. Web Content Accessibility Guidelines (WCAG) 2.1 define how to make web content more accessible to people with disabilities. Accessibility encompasses a wide range of disabilities, including visual, auditory, physical, speech, cognitive, language, learning, and neurological disabilities. The guidelines aim to make web content more usable for older individuals with changing abilities due to aging and to improve usability for users in general.

Executive summary

Manual evaluation was conducted on a sample of 32 pages and 5 screens without URLs.

The evaluation of the extent to which the website jumbo.com conforms to the Web Content Accessibility Guidelines (WCAG) 2.1 was completed on September 16, 2023. Only the WCAG 2.1 level A and AA success criteria from these guidelines were used in this evaluation, resulting in 22 out of 50 success criteria being met.

The quality of the subtitles in the tested video is very good. The text on the website can be resized to 200% without any loss of content or functionality. The page titles accurately describe the content of each page.

However, some accessibility issues were identified during the audit. These issues can be categorized into three groups:

  • Typos and minor code errors that can be easily rectified.
  • Excessive use of ARIA techniques. When not used properly, these techniques can introduce additional accessibility problems.
  • Some custom elements that require collective brainstorming on how to make them more accessible. For instance, the search bar or the moving buttons used to add products.

In the following chapter of this report, we will present all the accessibility issues that were identified. The examples cited in this report are drawn exclusively from the problems found in the sample. Consequently, this does not provide a comprehensive overview of all the existing issues. It is therefore possible that accessibility problems may still exist beyond the sample, which may be discovered in future evaluations when different samples are used.

Evaluation Findings
Decorative images

The large image on the homepage beneath the text 'Gratis eentje extra' is purely decorative. Decorative images should be concealed from assistive technology by leaving the 'alt' attribute of the image empty (alt=""). However, the 'alt' attribute of this image is set to 'Gezellig samen genieten'. Furthermore, at the bottom of this section, there are three blocks containing headings, images, and buttons. The 'alt' text of each image duplicates the text of the heading. These images are also decorative. To eliminate redundancy and hide these images from screen readers, please leave the 'alt' attributes empty.

On the homepage, beneath the heading 'Ontdek onze handige lijstjes,' there are shopping lists. Each item includes an image, the name of the shopping list, the number of followers, and the Jumbo logo represented as an image with a single letter 'J' along with the adjacent text 'Jumbo.' Presently, the logo has a text alternative that reads 'Jumbo profile image.' However, for all six items, the screen reader currently reads: 'Jumbo profile image Jumbo.' To resolve this, it is advisable to leave the 'alt' attribute of this logo empty to hide it from the screen reader. Generally, logos are considered informative images and should not be hidden, but in this specific situation, there is a text alternative provided as plain text on the page next to the logo, making it redundant for screen reader users. This small version of the logo is seen on other pages as well. See for example https://www.jumbo.com/recepten/pikante-pastasalade-met-broccoli-ansjovis-en-mozzar ella-503234.

In the footer of the website, there is a photo of a girl with a headset. This image is purely decorative and should not be interpreted by a screen reader. Typically, the way to achieve this is by leaving the 'alt' attribute empty, which is not the current configuration. Currently, the 'alt' attribute is set to 'klantenservice.' ARIA attributes can also be used to hide the image, such as 'aria-hidden="true"' and 'role="presentation".'

The SVG in Google Play logo in the footer of the website is not hidden from the screen reader. To solve this add aria-hidden=’true’.

On the page https://www.jumbo.com/service, under the section 'Actueel,' there is a another photo of a girl with a headset. Informative images

On the homepage, within the section featuring a special offer, there is a prominent image displaying the current special offer with an alt-text that reads 'Jumbo Melon.' This image and its alt-text may change weekly. The crucial information from the image like the price, conditions, and date are not accessible to screen reader users. To address this issue, consider placing the most vital information, such as the offer's price, conditions, and date, within the alt-text of the image. Another adequate solution is to add invisible screen reader text under the heading on the left.  It's advisable to remove the 'aria-label' attribute from the link element that functions as a container for the image. This modification ensures that screen reader users receive important details directly from the alt-text, making the special offer information more accessible and up-to-date.

In the footer of the website, there is a row with logos. All of these informative images are appropriately accompanied by text alternatives repeating the visible text. However, the logo of 'Amex' has a text alternative that reads 'American Express.' It is necessary to include the same text in the text alternative as what is represented in the logo itself. This ensures that blind visitors receive equivalent information to that of sighted visitors, promoting accessibility and inclusivity.

In the footer, the 'NIX18' logo is an informative image that currently lacks a text alternative. Additionally, the inline SVG is not hidden from the screen reader and is missing the 'role="img".' Consequently, certain screen readers may interpret the SVG code as text. To rectify this, it's advisable to add 'role="img"' to the inline SVG and provide an 'aria-label="NIX18"' attribute. This ensures that screen readers properly recognize the image and convey the informative label 'NIX18' to users with visual impairments.

There are certain concerns regarding the quality of the 'alt' texts used on the website. It's essential to ensure that these texts are written in the same language as the page's content because screen readers follow the pronunciation rules set at the HTML level. Currently, some buttons and aria-labels contain English text, even though the primary language of the page is Dutch. For instance, this includes the 'close' button for closing the mobile navigation and aria-labels on inline SVGs. To improve accessibility and language consistency, these elements should be updated to use Dutch text, aligning with the page's primary language. This ensures a more seamless and user-friendly experience for Dutch-speaking users relying on screen readers.

Evaluation Findings

Success criteria 1.2.3 and 1.2.5 both related to audio content on websites, but they address slightly different aspects of accessibility. Success criterion 1.2.3 has a broader scope, covering both audio and video content, and any visual information within these multimedia elements. On the other hand, success criterion 1.2.5 specifically focuses on the requirement for audio descriptions in pre-recorded video content.

Regarding videos, under 1.2.3, website creators have the flexibility to choose between providing a transcript or an audio description as alternatives for visual information. However, under 1.2.5, the provision of an audio description is mandatory when dealing with pre-recorded video content.

Why would you decide to add transcripts? Transcripts are text-based documents that serve as an alternative to information presented in an audible and visual format. A transcript is the only means of access to the information presented in the video for people who are deafblind. But there are other benefits to providing transcripts with audio and video content as well. Transcripts allow all users to browse through audio and video information at their own pace. Users can also search through the text and interact with the text (e.g., copy and paste text). Because transcripts are text-based, they are Search Engine Optimization (SEO)-friendly since search engines can crawl through the transcript text.

Currently, neither a text description nor audio description is available for the visual content. An example of videos’ without transcript is https://www.jumbo.com/recepten/pikante-pastasalade-met-broccoli-ansjovis-en-mozzar%20ella-503234. Put the information about the ingredients (at the beginning of the video) and the link to the recipe video (at the end of the video)  in the transcript.

Headings

On the homepage, there are headings that contain a <strong> element within an <h> element. This usage is semantically incorrect. The <strong> element should only be employed to add emphasis to text, not within heading elements. For instance, see the <h3> heading with the text "Gratis eentje extra" for an example of this issue. This issue is seen on every page of the sample.

On the homepage, the heading levels are arranged in a disorderly manner, leading to a lack of proper content structure. Immediately after the heading 3 'Ontdek onze handige lijstjes', there are heading level 2, which disrupts the logical hierarchy of headings. Change this order of headings. There are more examples of headings where the levels don’t make sense. See all italicized headings like ‘Da's mooi meegenomen!’ In later versions of the website the italicezed headings were gone.

HTML heading elements must accurately represent the visual structure and hierarchy of the content. This ensures that blind users can comprehend the relationship between different sections of the content.

On the page https://www.jumbo.com/account/mijn/jumbo the heading structure is not logical. The h1 element is followed by h5 elements.

On the page https://www.jumbo.com/aanbiedingen/actieprijs the heading level 5 ‘Tot en met dinsdag 5 sep’ is placed directly under h2 ‘In de actie’. H5 should not be a heading.

On the page https://www.jumbo.com/checkout/reserveren/wanneer, the side panel with ‘Kies een Jumbo winkel’ has a collection of shops. Each shop has a heading level 2 and the address, marked up as a heading level 3. The latter is not correct. Replace heading level 3 with a paragraph element.

On page https://www.jumbo.com/recepten headings 5 (the titles of the recipes ) are standing directly under heading 2 ‘Deze week op het menu onder €2 p.p.’. Under the heading level 3 ‘ Veel gestelde vragen over eten en gerechten’ there are questions. Each question is markup as a heading level 3.  Use h4 instead.
